# Facebook Blog Draft 3 - Demo Scope and Lessons

The biggest lesson from V-Trips is scope control. The original idea could become a large travel marketplace, but the deadline required a focused fullstack demo that actually runs.

We prioritized working breadth:
- place discovery and detail
- trip planning
- saved places
- reviews and ratings
- booking requests
- business/admin dashboards
- export action
- AI suggestion fallback
- AWS-ready documentation

We intentionally left production-depth items for future work: real Cognito login, Bedrock AI, realtime chat, advanced moderation, claim listing, Redis/OpenSearch, and CI/CD.

This makes the final report clearer: what works locally today, what is prototype/fallback, and what the AWS production path would be after submission.
